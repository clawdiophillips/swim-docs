# Training Session Workflow

Training footage is different from race footage — sessions are longer, footage is less structured, and the insights you want are more technical. This workflow covers how to get the most from SwimAI in a training context.

---

## What to Film in Training

You don't need to film entire sessions. Target specific sets where you want data:

| What to film | Why |
|---|---|
| **Time trial sets** | Compare splits to race benchmarks |
| **Threshold sets** | Track pacing consistency across reps |
| **Turn drills** | Detailed underwater analysis |
| **Starts** | Reaction time and first 15m |
| **Technique swims** | Side or underwater angle for breakout/stroke analysis |

<Callout type="info">
  Underwater or side-angle footage (filmed through the window or from beside the lane) gives much richer technique data than end-on footage. Use both angles where you have access.
</Callout>

---

## Uploading Training Videos

Training videos are often longer and less labelled than race footage. When uploading:

1. Use the **Title** field to include the set type: "Threshold 8×100 – Lane 3 – March session"
2. Set **Pool length** accurately — this affects split calculations
3. Use **Tags** to group by athlete, phase (pre-season / taper), or technique focus
4. You can upload multiple angles of the same set and annotate them separately

---

## Annotating a Time Trial

For a time trial in training, annotate the same way as a race:

- Start → Turn(s) → Finish for splits
- Add Breakout and 15m markers for underwater metrics

Then **compare to the most recent race** using the [Comparison](/docs/features/comparison) feature to see if training splits are matching race performance.

---

## Annotating a Repeat Set

<Steps>

### Upload the full set video

A 8×100m set might be 25–30 minutes of footage. Upload the whole clip.

### Create one annotation per rep

In the annotate screen, use Custom markers with names like "Rep 1 Start", "Rep 1 Turn 1", etc. to distinguish reps within a single video.

### Export to spreadsheet

Export the splits table as CSV. In Excel or Google Sheets, each rep's data is in its own row — easy to chart pacing across the set.

</Steps>

<Callout type="tip">
  **Alternatively**, if you have time to film each rep separately, upload separate files per rep. This keeps each annotation clean and makes comparison easier.
</Callout>

---

## Underwater and Technique Analysis

For detailed technique work, pair SwimAI with underwater footage:

1. Film through the observation window or with an underwater camera on a pole
2. Upload the angle you want to focus on — end view for turns, side view for stroke
3. Use **Custom markers** to tag stroke errors, body position changes, or technique cues
4. Use the **trim tool** to cut 5–10 second clips of specific moments to show the athlete

---

## Tracking Progress Over Time

SwimAI's library lets you see all videos of a given athlete side-by-side using the [Comparison](/docs/features/comparison) feature. A useful monthly habit:

1. Film a set of time-trial 50s or 100s at the start of each training block
2. Tag them by athlete and date
3. At the end of the block, compare the new time trial with the previous one
4. Look at changes in: total time, underwater time, turn cycle, first lap vs. back half

This gives athletes visible evidence of progress (or areas needing work) without relying on race results alone.
